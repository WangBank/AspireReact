# 心魔录项目文档

## 1. 项目概览

心魔录是一个用于个人投资复盘的全栈应用，核心目标是把“每天发生了什么”记录清楚，并把这些数据沉淀成可以回看、可以统计、可以写笔记的复盘资料。

项目目前覆盖四类核心数据：

- 账户日资金：总资产、持仓市值、可用资金、当日盈亏
- 银证转账：转入、转出、金额、备注
- 交易持仓明细：股票买卖、持仓数量、当日盈亏、累计盈亏、清仓状态
- 复盘笔记：全局笔记、按股票笔记

在此基础上，系统还提供：

- 图片识别导入
- 首页概览
- 统一数据列表
- 统计汇总
- 用户注册/登录/个人信息
- 同花顺链接配置

## 4. 运行方式

### 4.1 环境要求

- .NET SDK 10
- Node.js 20.19+ 或 22.12+
- PostgreSQL
- Redis

### 4.2 配置文件

后端主要配置文件：

- [AspireReact.Server/appsettings.json](AspireReact.Server/appsettings.json)

重要配置项：

| 配置项 | 作用 |
| --- | --- |
| `ConnectionStrings:PostgreSQL` | PostgreSQL 连接字符串 |
| `Redis:ConnectionString` | Redis 连接地址 |
| `Jwt` | 登录签发 Token 所需配置 |
| `RapidOcr` | OCR 模型目录和是否自动下载 |
| `Tonghuashun:LinkPrefix` | 股票详情链接前缀 |

### 4.3 启动方式一：Aspire 启动

在项目根目录执行：

```bash
dotnet run --project AspireReact.AppHost
```

说明：

- 前端默认对外端口为 `5516`
- AppHost 会同时拉起前端与后端
- 前端通过服务引用访问后端 API

推荐访问：

```text
http://localhost:5516
```

### 4.4 启动方式二：前后端分开启动

先启动后端：

```bash
dotnet run --project AspireReact.Server/AspireReact.Server.csproj --urls http://127.0.0.1:6202
```

再启动前端：

```bash
cd frontend
SERVER_HTTP=http://127.0.0.1:6202 npm install
SERVER_HTTP=http://127.0.0.1:6202 npm run dev
```

说明：

- 前端 Vite 会将 `/api` 请求代理到 `SERVER_HTTP`
- 适合单独调试前后端

### 4.5 OCR 模型说明

系统已集成 RapidOCRSharpOnnx。

当 `RapidOcr.AutoDownloadModels=true` 时：

- 首次识别截图会自动下载 OCR 模型
- 模型默认保存在 `AspireReact.Server/RuntimeData/RapidOcr`
- 首次识别耗时会明显高于后续识别

## 5. 页面与功能说明

### 5.1 登录页 `/login`

支持：

- 登录
- 注册
- 图形验证码

注册需要填写：

- 邮箱
- 用户名
- 密码
- 确认密码
- 验证码

### 5.2 首页 `/dashboard`

首页主要展示：

- 最近交易日期
- 最近交易日当日盈亏
- 今日 / 本周 / 本月 / 累计盈亏
- 最近账户记录
- 最近银证流水
- 最近交易记录
- 快捷录入入口

用途：

- 快速确认今天是否已经录入
- 快速查看最近一次录入对应的交易日期和盈亏

### 5.3 统一录入页 `/entry/unified`

这是当前推荐使用的主录入入口。

页面包含三块录入内容：

- 账户资金
- 银证转账
- 交易持仓

支持两种使用方式：

#### 方式 A：手工录入

适合每天逐项手动填写：

- 账户日期、总资产、持仓市值、可用资金、当日盈亏
- 银证转入/转出
- 多只股票的交易与持仓明细

#### 方式 B：图片识别导入

支持上传券商截图自动识别，并先回填表单，再由用户确认后入库。

当前支持的截图类型：

- 同花顺手机端持仓页整屏截图
- 含当日买入、当日卖出、买入均价、卖出均价、收盘价的流水表截图
- 左侧账户历史汇总 + 右侧当日流水明细的组合截图

识别后会尽量回填：

- 交易日期
- 账户总资产、持仓市值、可用资金、当日盈亏
- 银证流水转入/转出金额
- 股票代码、名称、板块
- 买卖数量、买卖均价、收盘价
- 持仓数量
- 当日盈亏、累计盈亏
- 是否清仓

统一录入页支持的识别相关操作：

- `识别并回填`
- `一键保存识别结果`
- `清空识别回填`
- 识别原图预览
- 识别提醒与 warning 展示

使用建议：

1. 先上传图片识别
2. 对照原图检查回填结果
3. 再点击一键保存

### 5.4 兼容录入页

这些页面仍保留，但不是当前主入口：

- `/entry/account`
- `/entry/bankflow`
- `/entry/trade`

用途：

- 单独编辑某一类数据
- 兼容旧入口或从列表页跳转编辑

### 5.5 统一数据列表 `/list/unified`

这是当前推荐使用的主列表页。

特点：

- 三种类型切换：账户列表 / 流水列表 / 交易列表
- 使用互斥单选切换，不使用下拉菜单
- 支持日期范围筛选
- 支持关键字筛选
- 支持刷新和重置
- 支持批量勾选删除

其中交易列表还有这些规则：

- 按每日分组显示
- 展示股票代码、名称、板块、状态、持仓盈亏、持仓数量、当日盈亏
- 状态明确区分“持仓”与“清仓”
- 编辑与删除在同一操作列

### 5.6 兼容列表页

这些页面仍可访问：

- `/list/account`
- `/list/bankflow`
- `/list/trade`

用途主要是兼容旧页面和局部调试。

### 5.7 统计页 `/statistics`

统计页当前支持的时间范围：

- 今日
- 本周
- 本月
- 本年
- 全部
- 自定义

盈亏筛选支持：

- 全部
- 盈利
- 亏损

统计卡片当前包括：

- 总盈亏
- 银证转账汇总
- 银证转入汇总
- 银证转出汇总
- 当前总额

表格当前包括：

- 按股票汇总的累计盈亏

适用场景：

- 看某个时间区间整体盈亏
- 看该区间内资金净流入/净流出
- 找出主要盈利或亏损股票

### 5.8 笔记页

#### 全局笔记 `/notes/global`

适合记录：

- 大盘复盘
- 当日纪律总结
- 交易计划

支持：

- 新建
- 编辑
- 删除
- 关键字搜索
- 日期筛选

#### 股票笔记 `/notes/stock`

适合记录：

- 某只股票的买入逻辑
- 卖出复盘
- 题材跟踪

支持：

- 按股票代码查看
- 搜索指定日期或关键字
- 新建 / 编辑 / 删除

### 5.9 设置页 `/config`

用于配置同花顺股票详情页链接前缀。

配置后：

- 交易列表中的股票代码可跳转
- 统计页中的股票代码可跳转

常见前缀：

- `https://www.10jqka.com.cn/`
- `https://stockpage.10jqka.com.cn/`

### 5.10 个人信息页 `/profile`

支持：

- 查看用户名、邮箱、注册时间、最后登录时间
- 修改用户名和邮箱
- 修改密码

## 6. 图片识别导入详细说明

### 6.1 接口

- `POST /api/portfolio-import/screenshot`

请求方式：

- `multipart/form-data`

主要字段：

| 字段 | 说明 |
| --- | --- |
| `image` | 上传图片文件 |
| `importDate` | 用户指定的导入日期 |

支持格式：

- PNG
- JPG
- JPEG
- WebP

大小限制：

- 10 MB

### 6.2 当前识别策略

后端识别服务会尝试：

- 解析右侧明细日期
- 匹配左侧账户汇总对应日期行
- 识别流水表头列位置
- 解析每日股票明细
- 自动标记清仓状态
- 对明细合计和顶部当日盈亏做校验

已做的容错包括：

- 手机端日期 OCR 粘连纠偏
- 表头别名与模糊匹配
- 收盘价与证券代码粘连修复
- 明细总和和顶部当日盈亏不一致时回退到顶部汇总值

### 6.3 使用建议

- 尽量保留完整表头
- 尽量不要裁掉右上角日期
- 组合截图最好同时保留左侧日期列和右侧流水区域
- 识别完成后先看回填结果和原图预览，再保存

## 9. 开发说明

### 9.1 当前主入口

当前导航默认使用的页面是：

- 录入：`/entry/unified`
- 数据列表：`/list/unified`

旧页面仍保留，主要用于：

- 兼容旧链接
- 从列表页跳转到单项编辑
- 局部调试

### 9.2 日期处理

项目已经针对本地日期显示做了修正，当前页面默认按本地日期格式展示，尽量避免前后端时区导致的“日期偏一天”问题。

### 9.3 股票搜索缓存

股票搜索当前使用：

- Redis
- PostgreSQL
- 第三方搜索接口

优先级为：

```text
Redis -> PostgreSQL -> 第三方接口
```

并带有简单限流，避免频繁请求搜索接口。

## 10. 常见使用建议

### 10.1 适合的日常流程

1. 早上或收盘后截一张券商图
2. 进入统一录入页识别并回填
3. 检查账户资金、银证转账和交易持仓
4. 一键保存
5. 在数据列表检查“持仓/清仓”状态
6. 到统计页看区间结果
7. 在笔记页补充当天复盘

### 10.2 如果识别结果不理想

优先检查：

- 图片是否包含完整表头
- 日期是否清晰可见
- 是否混入过多滚动遮挡或裁切
- 是否丢失左侧账户汇总或右侧明细区域

### 10.3 如果启动失败

优先检查：

- PostgreSQL 是否可连接
- Redis 是否已启动
- `appsettings.json` 连接串是否正确
- Node 和 .NET 版本是否满足要求
