## 📋 项目简介

**碎银录**是一个专为个人投资者设计的全栈Web应用，支持每日实盘交易数据的录入、统计、复盘和笔记管理。系统采用现代化的技术栈，具备高性能的心魔搜索缓存机制和完整的交易分析功能。

### 🎯 核心价值
- **个人实盘复盘**: 记录每日交易数据，支持多维统计分析
- **智能心魔搜索**: 集成Redis缓存+第三方API，实现毫秒级搜索
- **数据持久化**: PostgreSQL + Redis双存储，确保数据安全
- **现代化UI**: React + TypeScript + Vite，响应式设计

### 3. 心魔交易记录表 (`StockTrade`)
```sql
CREATE TABLE stock_trades (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    stock_code VARCHAR(10) NOT NULL,
    stock_name VARCHAR(50) NOT NULL,
    board INTEGER NOT NULL, -- C# 枚举 TradeBoard { 主板, 创业板, 科创板, 北交所 }
    buy_price DECIMAL(10,2) NOT NULL,
    buy_quantity INTEGER NOT NULL,
    sell_price DECIMAL(10,2) NOT NULL,
    sell_quantity INTEGER NOT NULL,
    position_pnl DECIMAL(18,2) NOT NULL,
    cumulative_pnl DECIMAL(18,2) NOT NULL,
    trade_note TEXT,
    tonghuashun_link TEXT
);
CREATE INDEX idx_stock_trades_trade_date_stock_code ON stock_trades(trade_date, stock_code);
```

CREATE TABLE trade_notes (

## 📝 编码规范

### 后端规范 (.NET)
1. **命名约定**
   - 实体类: `PascalCase` (如 `StockTrade`)
   - 数据库表: `snake_case` (如 `stock_trades`)
   - API端点: RESTful风格，使用复数名词
   - 异步方法: 后缀 `Async`

2. **项目结构**
   ```
   AspireReact.Server/
   ├── Controllers/          # API控制器
   ├── Services/             # 业务逻辑层
   ├── Entities/             # 数据库实体
   ├── Data/                 # DbContext/迁移
   ├── DTOs/                 # 数据传输对象
   ├── Middlewares/          # 中间件
   ```

3. **API响应格式**
   ```json
   {
     "success": true,
     "data": {...},
     "message": "操作成功",
     "timestamp": "2026-05-25T10:30:00Z"
   }
   ```

### 前端规范 (React + TypeScript)
1. **组件结构**
   ```

  },
  "Captcha": {
    "ExpiryMinutes": 5
  }

```

**前端配置** (`frontend/vite.config.ts`)
前端通过 Vite 代理将 `/api` 请求转发到后端，无需额外的环境变量文件 (.env)。代理目标由 Aspire 框架在运行时自动注入 `SERVER_HTTP` / `SERVER_HTTPS` 环境变量：
```typescript
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: process.env.SERVER_HTTPS || process.env.SERVER_HTTP,
        changeOrigin: true
      }
    }
  }
});
```
所有前端 Service 类直接使用相对路径 `/api/*` 发起请求，无需配置 API Base URL。
